# Episode 93 — "Zero hardcoded anything, and who gets the final word"
Author: Jason-eds · Draft 1 · ~2.5 min · Ed's 100 Rules

Delivery: Eddie, first person. Blank lines = card cuts. `[CARD: …]` = production cue.
Each episode pairs the day's **good rule** (top of the leaderboard) with a **bad rule**
(bottom) — the contrast is the hook.

- GOOD: Rule 20 — *Zero hardcoded values* (score 74.0, rank 3)
- BAD:  Rule 13 — *Five roles; human is final* (score 33.5, rank 98)

---

## 0:00 — The hook
[CARD: "74.0  vs  33.5"]

Third best rule I've got, against the ninety-eighth. One of them stops your code from
rotting the day the world changes around it. The other is the constitution of my
little crew. Same list, very different jobs.

## 0:20 — The good one
[CARD: "Rule 20 · Zero hardcoded values · 74.0"]

"Zero hardcoded values for anything that could plausibly change: hosts, ports, model
names, paths, timeouts, retry counts, feature flags, prompts."

Seventy-four — the top of the configuration section and third on the whole board. The
reason it scores so high is that hardcoding is the single most common way LLM-written
code goes wrong. The AI bakes a port, a model name, a timeout straight into a function
and now the code only runs in the exact world it was written in. Pull every one of
those out into config and the same binary runs on my laptop, in a container, in the
cloud — no edit. That's the difference between code you wrote and code you own.

## 1:10 — The bad one
[CARD: "Rule 13 · Five roles; human is final · 33.5"]

Now thirty-three and a half. "The crew is five fixed roles plus one human — Eddie.
Eddie's rulings are final and canonical: any persona's plan, preference, or pushback
yields to his decision." And there's the carve-out — Jason can push back when a new
ruling fights the canon.

It's a good governance principle. It's just *mine*. It scores low because it's about
the shape of my team, not about software. The "substitute your own name" note keeps it
from being pure trivia, but it's still organizational, not technical — it doesn't
generalize, and an AI can't really violate it the way it violates a hardcoding rule.

## 2:00 — The lesson
[CARD: "code that travels vs a team that's mine"]

That's the split. Rule twenty makes your code portable — it travels to any machine
anybody runs it on. Rule thirteen makes my *team* coherent, and only my team. A grade
isn't a value judgment on the rule; it's a measure of how far the rule reaches. One
reaches everybody. One reaches my desk.
